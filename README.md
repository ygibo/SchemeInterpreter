# C++ を用いて作成した Scheme インタプリター（未完成）
データ駆動設計を用いて作成している。そのため、パーサー（ScInterImpl/Parser/Parser.cpp) では、
自己評価式とシンボルとリストを構文解析して構文解析木を得ている。そして、define や if といった
scheme の構文要素は、リストの先頭の要素がそのキーワードだった場合、リストの残りの要素を引数とした、
define 等を実行する関数へ渡して実行する。そのキーワードと構文要素の関連付けは、
ScInterImpt/ScInter.cpp の setGlovalFunc 関数で行っている。そのため、
新しく構文要素を追加するには、その構文要素を実行するクラスを作り、そのクラスのオブジェクトを
キーワードに対応付けるだけで完了する。

現在、実装済みのものは以下のものとなる。各項目の右は例
整数値 16、文字列 "aaa"、ブール値 #t、文字 #\c、シンボル hoge、ペアー(a . b)、
バッククォート 'a、リスト (abc 16 "aaa")　以上のものが構文解析しているもの。
追加拡張されたものが、
define, list, lambda, set!, list、cons、quote、+、-、*、/、car、cdr となる。

if、cond やベクター、let、case、and、or、比較演算子等はまだ未実装
エラー処理は簡易的なものが少し入っている。基本的に正しい式を入力すると正しい結果を返すが、
そのうち、エラー処理を追加する
